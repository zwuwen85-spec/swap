```
┌──────────────────────────────────────────────────────┐
│                  Handler 层                           │

└──────────────────────────────────────────────────────┘
```

**各层职责：**

- **Handler层**：处理HTTP请求，参数绑定，响应封装
- **Logic层**：业务逻辑，事务控制，缓存策略
- **Model层**：数据库操作，SQL构建

### 3.3 交换模块

**交换状态机：**

```
┌─────────┐
│ pending │ 待处理
└────┬────┘
     │
     ├────────────→ accepted ──→ completed
     │               已接受        已完成
     │
     └────────────→ rejected
                     已拒绝

     └────────────→ cancelled
                     已取消
```

**交换流程：**

```
用户A发起交换请求
    ↓
创建exchange记录（status=pending）
    ↓
发送通知给用户B
    ↓
用户B操作：
    ├─ accept → status=accepted → 双方确认 → 完成
    ├─ reject → status=rejected → 记录原因
    └─ cancel → status=cancelled
```

### 3.4 WebSocket实时通信

**连接管理：**

```go
type Client struct {
    ID     int64
    Send   chan []byte
    Hub    *Hub
}

type Hub struct {
    Clients    map[int64]*Client  // 在线用户
    Register   chan *Client
    Unregister chan *Client
    Broadcast  chan *Message
}

func (h *Hub) Run() {
    for {
        select {
        case client := <-h.Register:
            h.Clients[client.ID] = client
            // 广播上线消息

        case client := <-h.Unregister:
            delete(h.Clients, client.ID)
            // 广播下线消息

        case message := <-h.Broadcast:
            // 转发消息给目标用户
        }
    }
}
```

**消息推送：**

```
客户端连接 → 验证Token → 加入Hub
    ↓
接收消息 → 验证格式 → 存储到MySQL
    ↓
查询接收者是否在线
    ├─ 在线：WebSocket推送
    └─ 离线：存入数据库，上线后拉取
```

### 5.2 缓存更新策略

**Cache-Aside模式：**

```
读取：
  查询缓存 → 命中返回 → 未命中查询DB → 写入缓存 → 返回

写入：
  更新DB → 删除缓存（而非更新，避免并发问题）
```

**示例代码：**

```go
func GetUser(userID int64) (*User, error) {
    // 1. 查询缓存
    key := fmt.Sprintf("user:info:%d", userID)
    cached, err := redis.Get(key).Result()
    if err == nil {
        var user User
        json.Unmarshal([]byte(cached), &user)
        return &user, nil
    }

    // 2. 查询数据库
    var user User
    if err := db.Where("id = ?", userID).First(&user).Error; err != nil {
        return nil, err
    }

    // 3. 写入缓存
    data, _ := json.Marshal(user)
    redis.Set(key, data, 1*time.Hour)

    return &user, nil
}

func UpdateUser(user *User) error {
    // 1. 更新数据库
    if err := db.Save(user).Error; err != nil {
        return err
    }

    // 2. 删除缓存
    key := fmt.Sprintf("user:info:%d", user.ID)
    redis.Del(key)

    return nil
}
```

### 6.2 并发优化

** Goroutine池：**

```go
type WorkerPool struct {
    taskChan chan func()
    wg       sync.WaitGroup
}

func NewWorkerPool(size int) *WorkerPool {
    pool := &WorkerPool{
        taskChan: make(chan func(), size*2),
    }

    for i := 0; i < size; i++ {
        pool.wg.Add(1)
        go pool.worker()
    }

    return pool
}

func (p *WorkerPool) worker() {
    defer p.wg.Done()
    for task := range p.taskChan {
        task()
    }
}

func (p *WorkerPool) Submit(task func()) {
    p.taskChan <- task
}
```

```go
func GetUserByID(ctx context.Context, userID int64) (*User, error) {
    var user User

        return nil, err
```
