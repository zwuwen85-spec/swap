# 举报模块测试指南

## 测试环境准备

1. 确保后端服务已启动
2. 确保数据库服务正常
3. 准备测试账号、商品、用户、评论数据

## 1. 创建举报测试

### 1.1 举报商品（正常流程）

**测试步骤：**
1. 用户A登录
2. 进入商品详情页（商品ID: 123）
3. 点击"举报"按钮
4. 选择举报原因："虚假信息"
5. 输入详细说明："该商品描述与实物不符"
6. 提交举报

**预期结果：**
- 提示"举报成功，我们会尽快处理"
- 数据库report表新增记录
- target_type=1, target_id=123, status=0

**验证数据库：**
```sql
SELECT * FROM report WHERE reporter_id = ? AND target_type = 1 AND target_id = 123;
```

### 1.2 举报用户

**测试步骤：**
1. 进入用户个人中心
2. 点击"举报"按钮
3. 选择举报原因："欺诈行为"
4. 输入详细说明
5. 提交举报

**预期结果：**
- 举报成功
- target_type=2

### 1.3 举报评论

**测试步骤：**
1. 在商品详情页找到某条评论
2. 点击评论的"举报"按钮
3. 选择举报原因："恶意攻击"
4. 提交举报

**预期结果：**
- 举报成功
- target_type=3

### 1.4 重复举报

**测试步骤：**
1. 用户A对商品123发起举报（待处理）
2. 再次对商品123发起举报

**预期结果：**
- 返回错误："您已经举报过该对象，请勿重复举报"
- 不能重复举报

### 1.5 举报不存在对象

**测试步骤：**
1. 调用API：POST /api/v1/report/create
2. 参数：{"target_type": 1, "target_id": 99999, "reason": "测试"}

**预期结果：**
- 返回错误："商品不存在"
- HTTP状态码：400

### 1.6 参数验证

**测试步骤：**
1. 不选择举报原因
2. 直接提交

**预期结果：**
- 前端验证：提示"请选择举报原因"
- 或后端验证返回参数错误

## 4. 处理举报测试

### 4.1 举报成立 - 商品

**测试步骤：**
1. 管理员调用API：PUT /api/v1/report/handle/1
2. 参数：{"status": 1, "handle_result": "确认违规，已下架"}

**预期结果：**
- 举报状态变为1（已处理）
- 记录处理人和处理结果
- **商品自动下架**（goods.status=0）
- 举报者可能收到通知

**验证数据库：**
```sql
-- 举报状态
SELECT status, handler_id, handle_result FROM report WHERE id = 1;
-- 商品状态
SELECT status FROM goods WHERE id = (SELECT target_id FROM report WHERE id = 1);
```

### 4.2 举报成立 - 用户

**测试步骤：**
1. 处理用户举报，status=1

**预期结果：**
- **用户自动冻结**（user.status=2）
- 该用户无法登录和发布商品

### 4.3 举报成立 - 评论

**测试步骤：**
1. 处理评论举报，status=1

**预期结果：**
- **评论自动隐藏**（comment.status=0）
- 评论不再显示

### 4.4 举报驳回

**测试步骤：**
1. 管理员调用API：PUT /api/v1/report/handle/2
2. 参数：{"status": 2, "handle_result": "经核实不违规"}

**预期结果：**
- 举报状态变为2（已驳回）
- 目标对象不受影响
- 记录驳回原因

### 4.5 重复处理

**测试步骤：**
1. 举报已经是已处理状态
2. 再次尝试处理

**预期结果：**
- 返回错误："该举报已被处理"
- 不能重复处理

### 4.6 处理权限验证

**测试步骤：**
1. 普通用户尝试处理举报

**预期结果：**
- 实际应用中应该有权限验证
- 当前简化版本需要登录

## 常见问题排查

### 问题1：举报后对象未处理

**可能原因：**
- HandleReport逻辑未执行
- 状态更新失败
- 事务问题

**解决方法：**
```sql
-- 检查举报状态
SELECT * FROM report WHERE id = ?;

-- 检查目标对象状态
-- 如果target_type=1
SELECT status FROM goods WHERE id = ?;
-- 如果target_type=2
SELECT status FROM user WHERE id = ?;
-- 如果target_type=3
SELECT status FROM comment WHERE id = ?;
```

### 问题2：重复举报成功

**可能原因：**
- 唯一性检查失效
- 并发问题

**解决方法：**
- 添加唯一索引：reporter_id + target_type + target_id
- 使用事务保证原子性

### 问题3：撤销按钮不显示

**可能原因：**
- 状态判断错误
- 权限验证问题

**解决方法：**
- 确认status=0时显示
- 确认reporter_id匹配当前用户
