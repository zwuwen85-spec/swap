## 交换状态流转

```
┌─────────┐
│ pending │ 待处理（初始状态）
└────┬────┘
     │
     ├────────────→ accepted ──→ completed
     │               已接受        已完成
     │
     └────────────→ rejected
                     已拒绝

     └────────────→ cancelled
                     已取消
```

### 状态说明

| 状态 | 值 | 说明 |
|-----|---|------|
| 待处理 | 0 | 初始状态，等待目标用户处理 |
| 已接受 | 1 | 目标用户接受，等待完成 |
| 已拒绝 | 2 | 目标用户拒绝，可填写拒绝原因 |
| 已取消 | 3 | 发起人主动取消 |
| 已完成 | 4 | 双方确认完成，商品状态变更为已交换 |

3. 点击"我想要"或"发起交换"
4. 选择类型为"购买"
5. 可选填写留言
6. 提交
7. ✅ 应该显示"交换请求已发送"

### 3. 测试发起交换（物物交换）

1. 使用用户A登录
2. 确保用户A有发布的商品
3. 浏览到用户B的商品
4. 选择类型为"交换"
5. 选择自己的商品
6. 提交
7. ✅ 应该显示"交换请求已发送"

### 4. 测试处理交换请求

使用用户B登录：

**接受交换：**
1. 访问"我的交换"页面
2. 查看"收到的请求"
3. 点击某个请求的"接受"按钮
4. 确认操作

**拒绝交换：**
1. 点击某个请求的"拒绝"按钮
2. 填写拒绝原因（可选）
3. 确认操作
4. ✅ 状态变为"已拒绝"

### 5. 测试取消交换

使用用户A（发起人）：
1. 访问"我的交换"页面
2. 查看"我发起的"
3. 找到待处理的请求
4. 点击"取消"按钮
5. 确认操作
6. ✅ 状态变为"已取消"

### 6. 测试完成交换

1. 找到已接受的交换请求
2. 双方确认后，点击"完成交换"按钮
3. 确认操作
4. ✅ 状态变为"已完成"
5. ✅ 相关商品状态变为"已交换"

## 数据库验证

查看交换记录：

```sql
USE campus_swap;

-- 查看所有交换记录
SELECT
  e.id,
  e.initiator_id,
  e.target_id,
  e.type,
  e.status,
  e.create_time,
  u1.username AS initiator_name,
  u2.username AS target_name,
  g.title AS goods_title
FROM exchange e
LEFT JOIN user u1 ON e.initiator_id = u1.id
LEFT JOIN user u2 ON e.target_id = u2.id
LEFT JOIN goods g ON e.goods_id = g.id
ORDER BY e.create_time DESC;
```

### 核心功能

**交换类型：**
1. **购买**：直接购买商品
2. **交换**：用我的商品交换你的商品

**权限控制：**
- 发起人：可以取消、完成
- 目标用户：可以接受、拒绝、完成
- 其他用户：只能查看

**业务规则：**
- 不能给自己的商品发起交换
- 同一个商品同一时间只能有一个待处理的请求
- 只有已接受的交换才能完成
- 完成交换后，相关商品状态变为"已交换"

**状态流转：**
```
pending (待处理)
    ↓
accepted (已接受)
    ↓
completed (已完成)

pending → rejected (已拒绝)
pending → cancelled (已取消)
```

## 常见问题

### Q: 为什么不能给自己的商品发起交换？

A: 这是业务逻辑限制，交换必须在不同用户之间进行。

### Q: 发起交换时提示"已有待处理的交换请求"？

A: 你已经向该商品发起过交换且还未处理完成，需要等待对方处理或取消后再发起。

### Q: 什么是"物物交换"？

A: 就是两个商品直接交换，不涉及金钱。选择"交换"类型时需要选择你的商品。

### Q: 交换完成后可以撤销吗？

A: 不可以。交换完成后状态不可逆，商品状态已变更为"已交换"。

### Q: 如何查看我发起的所有交换？

A: 在"我的交换"页面，选择"我发起的"即可查看。

## 注意事项

1. **交换完成后**
   - 商品状态自动变为"已交换"
   - 不可再进行交易
   - 建议双方进行线下交易

2. **权限验证**
   - 所有接口都需要登录
   - 只能查看自己参与的交换
   - 只能操作自己的请求

3. **状态管理**
   - 已完成的交换不可逆
   - 已拒绝/取消的请求不可恢复
   - 建议在操作前仔细确认

4. **用户体验**
   - 待处理数量提醒
   - 清晰的状态显示
   - 操作前二次确认
