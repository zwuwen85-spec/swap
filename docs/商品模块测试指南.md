# 商品模块测试指南

## 功能说明

商品模块已完成以下功能：

### 后端接口

1. **获取分类列表**
   - 接口：`GET /api/v1/category/list`
   - 返回：所有启用的分类

2. **获取商品列表**
   - 接口：`GET /api/v1/goods/list`
   - 参数：`page`, `page_size`, `category_id`, `type`, `condition`, `keyword`, `sort`
   - 返回：商品列表和总数

3. **获取商品详情**
   - 接口：`GET /api/v1/goods/detail?id={goods_id}`
   - 返回：商品详细信息

4. **发布商品**
   - 接口：`POST /api/v1/goods/create`
   - 需要认证：是
   - 参数：商品信息

5. **更新商品**
   - 接口：`PUT /api/v1/goods/update`
   - 需要认证：是

6. **删除商品**
   - 接口：`DELETE /api/v1/goods/delete`
   - 需要认证：是

7. **获取我的商品**
   - 接口：`GET /api/v1/goods/my`
   - 需要认证：是

8. **搜索商品**
   - 接口：`GET /api/v1/goods/search?q={keyword}`
   - 返回：搜索结果

9. **上传图片**
   - 接口：`POST /api/v1/upload/image`
   - 需要认证：是
   - 参数：`file`（multipart/form-data）

10. **批量上传商品图片**
    - 接口：`POST /api/v1/upload/goods-images`
    - 需要认证：是
    - 参数：`files`（最多9张）

### 前端页面

1. **商品列表页** (`/goods`)
   - 分类筛选
   - 类型筛选（售卖/交换/均可）
   - 成色筛选
   - 排序（时间/价格）
   - 关键词搜索
   - 分页加载

2. **商品详情页** (`/goods/:id`)
   - 图片轮播
   - 商品信息展示
   - 发布者信息
   - 我想要/发起交换/收藏

3. **发布商品页** (`/publish`)
   - 完整的表单
   - 图片上传（最多9张）
   - 表单验证

## 测试步骤

### 1. 准备测试数据

首先需要确保数据库中有分类数据。执行初始化SQL脚本会自动创建分类。

### 2. 测试商品列表

1. 访问：`http://localhost:3000/goods`
2. 应该显示商品列表（如果有数据）
3. 测试筛选功能：
   - 选择分类
   - 选择类型
   - 选择成色
   - 切换排序方式
4. 测试搜索功能：
   - 输入关键词搜索

### 3. 测试发布商品

1. 确保已登录
2. 访问：`http://localhost:3000/publish`
3. 填写商品信息：
   - 标题：测试商品
   - 描述：这是一个测试商品
   - 选择分类：数码产品
   - 选择类型：售卖
   - 输入价格：99.99
   - 选择成色：九成新
   - 上传图片（1-9张）
4. 点击"立即发布"
5. ✅ 应该显示"发布成功"并跳转到商品列表

### 4. 测试商品详情

1. 在商品列表中点击任意商品
2. 进入商品详情页
3. 查看商品信息是否正确显示
4. 测试图片预览功能
5. 测试"我想要"、"收藏"按钮

### 5. 使用Postman测试接口

#### 获取分类列表

```bash
curl -X GET http://localhost:8080/api/v1/category/list
```

#### 获取商品列表

```bash
curl -X GET "http://localhost:8080/api/v1/goods/list?page=1&page_size=20"
```

#### 获取商品详情

```bash
curl -X GET "http://localhost:8080/api/v1/goods/detail?id=1"
```

#### 发布商品（需要Token）

```bash
curl -X POST http://localhost:8080/api/v1/goods/create \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "测试商品",
    "description": "这是一个测试商品",
    "category_id": 1,
    "type": 1,
    "price": 99.99,
    "images": ["http://example.com/image.jpg"],
    "condition": 2
  }'
```

#### 上传图片（需要Token）

```bash
curl -X POST http://localhost:8080/api/v1/upload/image \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/image.jpg"
```

## 数据库验证

查看商品数据：

```sql
USE campus_swap;

-- 查看所有商品
SELECT id, title, category_id, type, price, status, create_time
FROM goods
ORDER BY create_time DESC
LIMIT 10;

-- 查看商品和分类
SELECT
  g.id,
  g.title,
  c.name AS category_name,
  g.price,
  g.status
FROM goods g
LEFT JOIN category c ON g.category_id = c.id
ORDER BY g.create_time DESC;
```

## 功能特性

### 已实现

✅ 商品数据模型（支持图片JSON存储）
✅ 分类管理
✅ 商品列表（分页、筛选、排序）
✅ 商品详情（浏览次数统计）
✅ 发布商品（多图上传）
✅ 图片上传到本地
✅ 我的商品管理
✅ 商品搜索
✅ 响应式布局

### 核心功能

**商品类型：**
- 售卖：只卖不换
- 交换：只换不卖
- 均可：既能卖也能换

**商品状态：**
- 0：下架
- 1：在售
- 2：已售
- 3：已交换

**图片处理：**
- 支持jpg、jpeg、png、gif格式
- 单张最大5MB
- 最多9张图片
- 自动生成唯一文件名

**分页与排序：**
- 默认每页20条
- 支持按时间/价格排序
- 支持多种筛选条件组合

## 注意事项

1. **图片存储路径**
   - 默认保存在 `backend/uploads/` 目录
   - 按日期分目录存储
   - 静态文件通过 `/uploads` 路径访问

2. **图片URL**
   - 上传成功后返回相对路径
   - 前端需要拼接完整URL
   - 生产环境建议使用CDN

3. **数据验证**
   - 所有输入都经过验证
   - 售卖类型必须填写价格
   - 图片至少上传一张

4. **性能优化**
   - 商品列表支持分页
   - 图片使用懒加载
   - 建议添加Redis缓存

---

**提示**：发布商品时，图片是必需的。如果图片上传失败，请检查：
1. `uploads` 目录是否有写权限
2. 文件大小是否超过限制
3. 文件格式是否正确
