# 收藏模块测试指南

## 测试环境准备

1. 确保后端服务已启动
2. 确保数据库服务正常
3. 准备测试账号和商品数据

## 1. 添加收藏测试

### 1.1 添加收藏（正常流程）

**测试步骤：**
1. 用户A登录系统
2. 进入商品详情页面（商品ID: 123）
3. 点击"收藏"按钮

**预期结果：**
- 按钮状态变为"已收藏"
- 按钮图标变为实心星标
- 显示成功提示："收藏成功"
- 商品收藏数+1
- 数据库favorite表新增一条记录

**验证数据库：**
```sql
SELECT * FROM favorite WHERE user_id = ? AND goods_id = ?;
SELECT favorite_count FROM goods WHERE id = ?;
```

### 1.2 重复收藏

**测试步骤：**
1. 用户A对已收藏的商品再次点击"收藏"按钮

**预期结果：**
- 显示错误提示："已经收藏过了"
- 数据库不新增记录
- 商品收藏数不重复增加

### 1.3 收藏不存在的商品

**测试步骤：**
1. 直接调用API：POST /api/v1/favorite/add
2. 参数：{"goods_id": 99999}

**预期结果：**
- 返回错误提示："商品不存在或已下架"
- HTTP状态码：400

### 1.4 收藏已下架商品

**测试步骤：**
1. 选择一个status=0（已下架）的商品
2. 尝试收藏

**预期结果：**
- 返回错误提示："商品不存在或已下架"
- 无法收藏已下架商品

### 1.5 未登录收藏

**测试步骤：**
1. 用户未登录状态
2. 访问商品详情页
3. 点击"收藏"按钮

**预期结果：**
- 提示："请先登录"
- 跳转到登录页面

## 2. 取消收藏测试

### 2.1 取消收藏（正常流程）

**测试步骤：**
1. 进入已收藏的商品详情页
2. 点击"已收藏"按钮

**预期结果：**
- 按钮状态变为"收藏"
- 按钮图标变为空心星标
- 显示成功提示："取消收藏成功"
- 商品收藏数-1

### 2.2 取消未收藏的商品

**测试步骤：**

**预期结果：**
- 返回错误提示："未登录或登录已过期"

## 常见问题排查

### 问题1：收藏数不准确

**可能原因：**
- 并发导致favorite_count更新失败
- 商品被删除但favorite记录未清理

**解决方法：**
- 使用数据库原子操作：`UPDATE goods SET favorite_count = favorite_count + 1 WHERE id = ?`
- 定期清理脏数据：删除孤立收藏记录
- 实现定时任务同步favorite_count

### 问题2：重复收藏成功

**可能原因：**
- 数据库唯一索引缺失

**解决方法：**
```sql
-- 添加唯一索引
ALTER TABLE favorite ADD UNIQUE KEY uk_user_goods (user_id, goods_id);
```

### 问题3：收藏列表为空

**可能原因：**
- user_id获取错误
- Preload关联查询失败

**解决方法：**
- 检查JWT token解析
- 确认user_id正确传递
- 检查GORM关联关系配置

## 测试数据清理

测试完成后清理测试数据：

```sql
-- 删除测试收藏记录
DELETE FROM favorite WHERE user_id IN (测试用户ID列表);

-- 重置商品收藏数
UPDATE goods SET favorite_count = 0 WHERE id IN (测试商品ID列表);

-- 或重新计算收藏数
UPDATE goods g SET favorite_count = (
  SELECT COUNT(*) FROM favorite f WHERE f.goods_id = g.id
);
```
