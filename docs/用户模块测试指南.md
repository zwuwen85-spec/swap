# 用户模块测试指南

## 功能说明

用户模块已完成以下功能：

### 后端接口

1. **用户注册**
   - 接口：`POST /api/v1/user/register`
   - 参数：`username`, `password`, `email`（可选）, `phone`（可选）
   - 返回：用户信息和JWT Token

2. **用户登录**
   - 接口：`POST /api/v1/user/login`
   - 参数：`username`, `password`
   - 返回：用户信息和JWT Token

3. **获取用户信息**
   - 接口：`GET /api/v1/user/info`
   - 需要认证：是
   - 返回：当前用户信息

4. **更新用户信息**
   - 接口：`PUT /api/v1/user/info`
   - 需要认证：是
   - 参数：`nickname`, `avatar`, `gender`, `phone`, `school`

5. **修改密码**
   - 接口：`POST /api/v1/user/change-password`
   - 需要认证：是
   - 参数：`old_password`, `new_password`

6. **上传头像**
   - 接口：`POST /api/v1/user/avatar`
   - 需要认证：是
   - 参数：`file`（multipart/form-data）

### 前端页面

1. **登录页面** (`/login`)
   - 用户名/密码登录
   - 跳转到注册页面
   - 登录成功后跳转到首页

2. **注册页面** (`/register`)
   - 用户名/密码/确认密码注册
   - 可选邮箱输入
   - 密码确认验证
   - 注册成功后跳转到首页

3. **个人中心** (`/profile`)
   - 显示当前登录用户信息
   - 退出登录功能

## 测试步骤

### 1. 启动数据库

确保MySQL和Redis正在运行：

```bash
# 检查MySQL
mysql -u root -p -e "SELECT 1"

# 检查Redis
redis-cli ping
```

### 2. 初始化数据库

```bash
cd D:\毕业设计\CampusSwapShop
mysql -u root -p < scripts/init.sql
```

### 3. 启动后端服务

```bash
cd backend
go run cmd/server/main.go
```

后端将在 `http://localhost:8080` 启动

### 4. 启动前端服务

```bash
cd frontend
npm install  # 首次运行需要
npm run dev
```

前端将在 `http://localhost:3000` 启动

### 5. 测试注册功能

1. 打开浏览器访问：`http://localhost:3000/register`
2. 输入用户名（3-20字符）
3. 输入密码（6-20字符）
4. 确认密码
5. 可选输入邮箱
6. 点击"注册"按钮
7. 应该显示"注册成功"并跳转到首页

### 6. 测试登录功能

1. 打开浏览器访问：`http://localhost:3000/login`
2. 输入刚才注册的用户名和密码
3. 点击"登录"按钮
4. 应该显示"登录成功"并跳转到首页

### 7. 测试API接口

使用Postman或curl测试：

#### 注册

```bash
curl -X POST http://localhost:8080/api/v1/user/register \
  -H "Content-Type: application/json" \
```

#### 登录

```bash

  -H "Content-Type: application/json" \
  -d "{\"username\":\"testuser\",\"password\":\"123456\"}"
```

预期响应：

## 常见问题

### 1. 数据库连接失败

**问题**：`Error 1045: Access denied for user 'root'@'localhost'`

**解决**：
- 检查 `config/config.yaml` 中的MySQL密码是否正确
- 确保MySQL服务正在运行

### 2. Redis连接失败

**问题**：`Redis connection failed`

**解决**：
- 确保Redis服务正在运行
- 检查 `config/config.yaml` 中的Redis配置

### 3. 用户名已存在

**问题**：注册时提示"用户名已存在"

**解决**：
- 使用不同的用户名
- 或删除数据库中的现有用户：
```sql
USE campus_swap;
DELETE FROM user WHERE username = 'testuser';
```

### 4. Token无效

**问题**：请求接口时提示"Token无效"

**解决**：
- Token有效期是7天，过期后需要重新登录
- 确保请求头格式正确：`Authorization: Bearer {token}`

### 5. CORS错误

**问题**：前端请求时出现CORS错误

**解决**：
- 确保后端CORS中间件已正确配置
- 检查前端代理配置是否正确
