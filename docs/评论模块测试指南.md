# 评论模块测试指南

## 测试环境准备

1. 确保后端服务已启动
2. 确保数据库服务正常
3. 准备测试账号、商品数据

## 1. 创建评论测试

### 1.1 创建主评论（正常流程）

**测试步骤：**
1. 用户A登录
2. 进入商品详情页（商品ID: 123，卖家为用户B）
3. 点击"写评价"按钮
4. 选择评分：4星
5. 输入评论内容："商品不错，卖家态度好"
6. 提交评论

**预期结果：**
- 提示"评价成功"
- 评论立即显示在评论列表中
- 数据库comment表新增记录
- goods_id为商品ID
- user_id为评论者ID
- target_user_id为卖家ID
- parent_id为0（主评论）
- rating为4

**验证数据库：**
```sql
SELECT * FROM comment WHERE goods_id = 123 AND user_id = ? AND parent_id = 0;
```

### 1.2 回复评论

**测试步骤：**
1. 用户B（卖家）登录
2. 进入同一商品详情页
3. 找到用户A的评论
4. 点击"回复"按钮
5. 输入回复内容："感谢您的好评！"
6. 提交回复

**预期结果：**
- 提示"回复成功"
- 回复显示在原评论下方
- parent_id为原评论ID
- target_user_id为原评论者ID

### 1.3 评论内容为空

**测试步骤：**
1. 不输入任何内容
2. 直接点击"发表"按钮

**预期结果：**
- 提示"请输入评价内容"
- 无法提交

### 1.4 评论内容超长

**测试步骤：**
1. 输入超过500字符的内容
2. 尝试提交

**预期结果：**
- 前端限制输入为500字符
- 或后端验证返回错误

### 1.5 评分边界测试

**测试步骤：**
1. 尝试提交评分0
2. 尝试提交评分6

**预期结果：**
- 参数验证失败
- 提示"评分必须在1-5之间"

### 1.6 评价不存在的商品

**测试步骤：**
1. 调用API：POST /api/v1/comment/create
2. 参数：{"goods_id": 99999, "content": "测试", "rating": 5}

**预期结果：**
- 返回错误："商品不存在"
- HTTP状态码：400

### 1.7 未登录评价

**测试步骤：**
1. 用户未登录
2. 点击"写评价"按钮

**预期结果：**
- 提示"请先登录"
- 跳转到登录页面

## 常见问题排查

### 问题1：评论不显示

**可能原因：**
- status为0（已删除）
- parent_id不为0（回复评论）
- 商品ID不匹配

**解决方法：**
```sql
-- 检查评论状态
SELECT * FROM comment WHERE goods_id = 123;
-- 确认status=1且parent_id=0
```

### 问题2：评分不准确

**可能原因：**
- 包含了回复评论的评分
- 统计逻辑错误

**解决方法：**
- 确保SQL只统计parent_id=0的评论
- 使用AVG和COUNT聚合函数

### 问题3：删除后评论仍显示

**可能原因：**
- 前端缓存未刷新
- 查询条件未过滤status

**解决方法：**
- 删除后刷新列表
- SQL添加WHERE status = 1条件
